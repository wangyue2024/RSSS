# RSSS: 项目模块划分及源文件结构分析 v3.0

本项目采用扁平化且高度组件化的工作区设计（单 Crate 多二进制架构）。代码集中在 `src/` 下，不同阶段的功能被严格划分到独立的模块中，保障了单一职责和极高的测试覆盖率。

## 核心架构大图

```text
┌───────────────── UI / Host App ─────────────────────────┐
│ src/main.rs (CLI / Args / TUI Boot / 3-Thread Setup)    │
│ src/tui.rs  (Ratatui 终端面板展示)                      │
│ src/record.rs (后台异步 CSV 写盘引擎)                   │
└───────────────────────┬─────────────────────────────────┘
                        │ spawn
┌───────────────── Simulation 主循环 ─────────────────────┐
│ src/simulation.rs                                       │
│ ├─ world.rs        上帝类：6 阶段仿真循环的入口         │
│ ├─ agent.rs        Agent 环境：Scope、持久化变量封装    │
│ ├─ settlement.rs   清结算：处理事件，调整资金与持仓     │
│ ├─ indicators.rs   滑窗计算引擎：增量 MA, VWAP, RSI 等  │
│ └─ config.rs       全局仿真参数定义                     │
└───────────────────────┬─────────────────────────────────┘
                        │ generate Events
┌───────────────── 撮合引擎 & BTreeMap ───┬───────────────┐
│ src/engine.rs                           │ src/scripting │
│ ├─ book.rs     双向订单簿，撮合与 GC    │ (桥接 Rhai 层)│
│ ├─ queue.rs    FIFO 同价队列，L2 汇总   │ api.rs        │
│ ├─ events.rs   领域协议：Trade, Cancel  │ math.rs       │
│                                         │ sandbox.rs    │
└───────────────────────┬─────────────────┴───────────────┘
┌───────────────── Domain 类型安全 ───────────────────────┐
│ src/domain.rs                                           │
│ ├─ types.rs    Price, Vol, Side, Order(32Bytes对齐)     │
│ └─ fixed.rs    10^6 缩放常数，安全运算函数              │
└─────────────────────────────────────────────────────────┘
```

## 1. 基础内核层 (`src/domain/`)
- 功能边界：定义所有最原子的不可变“名词”，**不依赖任何其余业务模块**。
- `fixed.rs`: 定义 `SCALING_FACTOR = 1_000_000`。提供防溢出的 `calculate_cost`, `calculate_fee`, `micros_to_display` 工具。绝不允许出现 `f64`。
- `types.rs`: 使用 NewType Pattern 封装 `Price(i64)` 和 `Vol(u64)`。保障编译期的类型绝对安全（不会错把价格加到量上）。定义了 `#[repr(C)]` 下 32 Byte 极度紧凑的 `Order` 结构体，优化 CPU 缓存命中（一个 64B Cache Line 正好放俩 Order）。

## 2. 金融撮合层 (`src/engine/`)
- 功能边界：纯粹的挂单吃单博弈引擎，不查账户，不管资金。接收 `Order`，返回 `MatchEvent`（撮合事件序列）。
- `book.rs`: 使用 `BTreeMap<Price, LevelQueue>` 作为核心高速树形盘口。并利用 `HashMap<u64, OrderMeta>` 充当倒排撤单索引（这是实现 O(1) **影子撤单** 的关键）。附带定期批量内存回收的 `gc_phantom_orders` 逻辑。
- `queue.rs`: `LevelQueue` 定义。保证同价位 FIFO（时间优先原则）。包含 `total_volume` 缓存，加速 `get_l2_snapshot` 获取深度的效率。
- `events.rs`: 核心通讯协议协定，定义 `MatchEvent::Trade` / `Placed` / `Cancelled` / `Rejected` 以及自我对敲拦截 `SelfTradeCancelled`。

## 3. 脚本融合桥接层 (`src/scripting/`)
- 功能边界：将复杂的 Rust 数据结构映射为 Rhai 脚本能无锁高效读取的内存。
- `api.rs`: `MarketState`, `AccountView` 的结构定义，以及向 `rhai::Engine` 注册的大量 `register_get` Getter 闭包函数。这里定义了 `ActionMailbox` 进行 Agent 的命令收集。
- `engine_builder.rs`: 工厂类，生成并配置全局统一且具备 `Send + Sync` 能力的 `rhai::Engine` 实例。
- `math.rs`: 以原生计算速度提供各类规避了浮点缺陷的技术算子（如 `arr_sum`, `arr_std_dev`, `arr_slope`）。
- `rng.rs`: `AgentRng` 确定性随机器。
- `sandbox.rs`: 提供 `compile_and_validate` 与 `DRY_RUN_TICKS` 多段沙盒防崩溃验证，限制单 Tick `MAX_OPERATIONS` 以防止 Agent 陷入死循环。

## 4. 仿真时空环境层 (`src/simulation/`)
- 功能边界：掌控全局的推演，“造物主”模块。
- `world.rs`: 承载最大的 `World` 结构，提供核心的 `run_tick()` 6 阶段推演（包括 Rayon 的多线程 `par_iter_mut()` 代理调度），收集订单，洗牌（Shuffle），并分发事件。
- `agent.rs`: 统筹单个实体状态 `AgentState`。维护他的 `Scope`、脚本抽象语法树 `AST` 以及 `locked_cash` 锁定状态和连错熔断机制。
- `settlement.rs`: 结算控制者。负责在 Phase 5（执行阶段）拦截非法的订单（`validate_action` 检测余额不足），并在撮合产生 `MatchEvent` 后执行扣钱，扣票，解冻（`settle_trade`, `update_order_books_*`）。
- `indicators.rs`: 高性能递增指标引擎。基于最新的报价增量，O(1) 更新 MA5/20/60、VWAP、ATR 和 RSI。

## 5. 支撑周边模块

### `src/record/`
数据持久化的后台 IO 引擎。
- 利用 MPSC (Multiple Producer Single Consumer) 的无锁/异步信道将主线程（`World`）和专门的写盘线程拆分开。主线程只需推数据包裹，不受慢速 IO (CSV 流化开销) 阻塞。

### `src/tui/`
强大的实时显示终端面板。
- 基于 `ratatui` 和 `crossterm`。从上报的加锁区读取轻量化的 `SharedUiState` 绘制：
- K线/价格历史轨迹。
- L2 十档完整深度树。
- 当前最赚钱的 Top 10 Agent 排行榜。
- 引擎实时 TPS (Ticks Per Second) 监控。

### `src/main.rs`
程序的 CLI 主干。负责：
- 接收命令行入参（`--agents`, `--ticks`, `--no-tui` 等）。
- 发起加载本地 `agent_generator/output` 目录下的大量 .rhai 脚本的动作。
- 拉起 3 条并行主线程的协作（Simulation, UI Render, IO Log）。
